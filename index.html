Como funciona? O que acontece com uma requisição feita para um site gerenciado pelo IIS?

O que é o IIS?

O IIS (Internet Information Services) é um servidor web criado pela Microsoft.

Um servidor web é responsável por receber e enviar pedidos HTTP de clientes (normalmente browsers) além de manter aplicações centralizadas com acessos de diferentes localizações.

O que acontece com uma requisição feita para um site gerenciado pelo IIS?

1. O browser inicia uma requisição HTTP para o servidor web e o HTTP.sys intercepta essa requisição.
2. O HTTP.sys consulta o WAS para obter as informações de configuração.
3. O WAS consulta informações de configurações no applicationHost.config.
4. O WWW service recebe as informações de configuração como application pool
5. O WWW service usa as informações de configurações para configurar o HTTP.sys.
6. O WAS inicia um worker process para o application pool que foi feita a requisição.
7. O worker process processa a requisição e retorna a resposta para o HTTP.sys.
8. O cliente recebe o response.

O que é o HTTP.sys?

Ele é um protocol listener (ouvinte de protocolo).
Todo protocol listener recebe um requisição de protocolo especifica, envia ele para o IIS para processamento e então retorna a resposta para quem fez a requisição.
Qualquer requisição do client para o servidor vai bater no HTTP.sys primeiro.

O que é o WWW service? WWW Service (World Wide Web Publishing Service)

• Responsável por gerenciar processos:
Gerencia o application pool e o worker process. Ele controla o início, pausa, recycle de um processo, além de monitorar e detectar falhas nos processos.

• Monitoria de performance:
Monitora a performance e oferece informações como o performance conter para os sites e o cache do IIS.

• Configuração/administração HTTP
Le as informações configuração e usa para configurar e atualizar o HTTP listener (HTTP.sys)

O que é o Listener adapter?

São componentes que estabelecem a comunicação entre o WAS e o protocol listener. Uma vez que o protocol adapter recebe as informações de configuração, ele configura seu protocol listener correspondente e prepara os listeners para ouvir os requests.

O que é o WAS? Windows Process Activation Service (WAS)

Usado Para expandir os usos de protocolos além do HTTP e HTTPS
Gerencia o application pool e o worker process tanto para requisições http e não http.
Logo no startup, o was lê as informações do arquivo ApplicationHost.config e passa as informações para os listeners adapter que estão no servidor.
Quando o applicationHost.config muda, o WAS recebe uma notificação e atualiza os adaptadores listeners com a nova configuração.
Obs. O was pode rodar sem utilizar o WWW Service, caso não precise utilizar as funcionalidades do http

O que é um WORKER PROCESS? (w3wp)

É um processo do Windows (w3wp.exe) que roda aplicações web.
